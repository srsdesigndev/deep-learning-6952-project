## 1. Stacking Multiple Feature Maps

In a real CNN, each convolutional layer doesn't just have one filter — it has **many filters**, and each filter produces its own feature map. The number of filters is a hyperparameter you choose.

**Key ideas:**
- Every "pixel" in a feature map corresponds to exactly one neuron
- All neurons in the same feature map share the same kernel (filter weights) and bias — this is **weight sharing**, and it's what keeps CNNs efficient
- Each feature map uses its own unique weights (different filters → different maps)
- The receptive field of a neuron in layer L extends across **all feature maps** of the previous layer — so each neuron sees depth as well as height/width
- Color images have 3 input channels (Red, Green, Blue), so the first conv layer actually processes 3 stacked layers

**Why this is powerful:** Because the same filter is applied across the entire image, the network can detect a feature (like an edge) regardless of where it appears. This is called **translation invariance**.

## 2. The Convolution Formula (Know What the Variables Mean)

The output of a neuron at position (i, j) in feature map k is:

```
z(i,j,k) = b_k + sum over receptive field of [ x(i', j', k') * w(u, v, k', k) ]
```

**What each variable means:**
- `z(i,j,k)` — output of neuron at row i, col j in feature map k
- `b_k` — bias for feature map k (like a brightness knob for that map)
- `x(i', j', k')` — input neuron from the previous layer at row i', col j', channel k'
- `w(u, v, k', k)` — the filter weight connecting input channel k' to output feature map k, at relative position (u, v) in the receptive field
- `s_h`, `s_w` — vertical and horizontal stride (step size of the sliding window)
- `f_h`, `f_w` — height and width of the receptive field (kernel size)

You won't need to compute this by hand on the exam, but know what each term represents.

## 3. Convolution By Hand (Example From Lecture)

Given filter `w = [[10, 10], [-1, -1]]` and a 4×4 input image, with stride=1:

Each output value is computed by sliding the 2×2 filter over the image and taking the dot product at each position. For example, the top-left output is:
```
10(4) + 10(3) + (-1)(-1) + (-1)(-1) = 40 + 30 + 1 + 1 = 72
```

The result is a smaller matrix (3×3 in this case, because no padding was used).

**Key takeaway:** Convolution = sliding the filter, multiplying element-wise, summing up.

## 4. Implementing Conv Layers in PyTorch

PyTorch expects image tensors in the format `[batch, channels, height, width]`.

```python
# Permute from [batch, H, W, channels] to [batch, channels, H, W]
images = images.permute(0, 3, 1, 2)

# Basic conv layer: 3 input channels (RGB), 32 filters, 7x7 kernel
conv_layer = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=7)

# With zero-padding so output size = input size
conv_layer = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=7, padding="same")

# With stride=2: output size is roughly halved
conv_layer = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=7, stride=2, padding=3)
```

**Weight shape:** For the above layer → `[32, 3, 7, 7]` = 32 filters × 3 input channels × 7×7 kernel
**Bias shape:** `[32]` = one bias per filter

**Important observation:** Randomly initialized filters already act like edge detectors. During training, the network learns better, task-specific filters automatically — you never design them by hand.

**Zero-padding side effect:** When using `padding="same"`, neurons near the border of the image can't fully "see" their receptive field, so zero values are substituted. This causes slightly weaker activations near the edges — visible as dark borders in the feature maps.

## 5. Pooling Layers

Pooling layers **downsample** the feature maps — they reduce height and width to save computation and memory. They have no learnable parameters.

**Why pool?**
- Reduces the spatial size of the representation → fewer computations in later layers
- Introduces **translation invariance** — small shifts in the input image don't change the output much

**Max Pooling** — Takes the maximum value in each region. Keeps the strongest activation. Most commonly used.

**Average Pooling** — Takes the average value in each region. Smoother output.

```python
# Max pool with 2x2 window (default stride = kernel_size, so output is halved)
max_pool = nn.MaxPool2d(kernel_size=2)

# Average pool
avg_pool = nn.AvgPool2d(kernel_size=2)
```

A 70×120 image after `MaxPool2d(kernel_size=2)` becomes 35×60.

## 6. Data Augmentation

When you don't have enough training images, you can **artificially create more** by applying realistic transformations to existing images. This is called **data augmentation** and is very commonly used in vision models.

**Common augmentation techniques:**
- Shifting (translating) the image
- Rotating
- Flipping horizontally/vertically
- Cropping to a different region
- Brightening or darkening
- Adjusting contrast or saturation
- Resizing
- Adding noise (random pixel disturbances)
- Adding blur

The key constraint is that the augmented image must still be **realistic and correctly labeled** — you wouldn't flip a digit "6" upside down and still call it a "6".

## 7. Full CNN Architecture Example (Fashion MNIST)

A typical CNN stacks: Conv → ReLU → Pool → Conv → ReLU → Pool → ... → Flatten → Dense → Output

```python
from functools import partial

DefaultConv2d = partial(nn.Conv2d, kernel_size=3, padding="same")

model = nn.Sequential(
    # First block: large kernel to capture broad features
    DefaultConv2d(in_channels=1, out_channels=64, kernel_size=7), nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),          # spatial size halved

    # Second block: two conv layers, then pool
    DefaultConv2d(in_channels=64, out_channels=128), nn.ReLU(),
    DefaultConv2d(in_channels=128, out_channels=128), nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),          # spatial size halved again

    # Third block: deeper features
    DefaultConv2d(in_channels=128, out_channels=256), nn.ReLU(),
    DefaultConv2d(in_channels=256, out_channels=256), nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),          # spatial size halved again

    # Dense head
    nn.Flatten(),                         # collapse 2D maps to 1D
    nn.Linear(2304, 128), nn.ReLU(),
    nn.Dropout(0.5),                      # regularization
    nn.Linear(128, 64), nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(64, 10),                    # 10 output classes
).to(device)

optimizer = torch.optim.AdamW(model.parameters())
```

**Pattern to remember:**
- Filter count increases as you go deeper: 64 → 128 → 128 → 256 → 256
- Spatial size decreases as you go deeper (due to pooling)
- Dropout is applied in the dense layers, not the conv layers
- Flatten converts the 2D feature maps into a 1D vector before the dense layers
- AdamW is used as the optimizer

**This model reached ~91% accuracy on Fashion MNIST after 20 epochs.**

## Key Concepts Summary

| Concept | What It Does |
|---------|-------------|
| Multiple filters per layer | Each filter learns a different feature; output = stack of feature maps |
| Weight sharing | All neurons in a feature map share the same filter → fewer parameters |
| Zero-padding | Keeps output same size as input; causes edge artifacts |
| Stride > 1 | Shrinks output size without pooling |
| Max pooling | Downsamples by keeping the max value in each region |
| Data augmentation | Artificially increases training data with realistic image variations |
| Flatten | Converts 2D feature maps to 1D vector before dense layers |
| Dropout in dense layers | Regularization to prevent overfitting |
