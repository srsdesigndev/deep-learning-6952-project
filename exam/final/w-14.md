## 1. CNN Architecture Overview (What to Know)

A typical CNN architecture follows this general flow:

```
Input Image → [Conv → ReLU → Pool] × N → Flatten → Dense Layers → Output
```

Each block of Conv+Pool progressively reduces the spatial size while increasing the depth (number of feature maps). The dense layers at the end do the actual classification.

The model summary from lecture shows this clearly for Fashion MNIST (28×28 grayscale input):

| Layer | Output Shape | What Happened |
|-------|-------------|----------------|
| Conv2d (7×7, 64 filters) | [1, 64, 28, 28] | 64 feature maps, same spatial size (padding="same") |
| MaxPool2d (2×2) | [1, 64, 14, 14] | spatial size halved |
| Conv2d (3×3, 128 filters) | [1, 128, 14, 14] | more filters, same size |
| Conv2d (3×3, 128 filters) | [1, 128, 14, 14] | stacked conv without pool |
| MaxPool2d (2×2) | [1, 128, 7, 7] | halved again |
| Conv2d (3×3, 256 filters) | [1, 256, 7, 7] | even more filters |
| Conv2d (3×3, 256 filters) | [1, 256, 7, 7] | stacked again |
| MaxPool2d (2×2) | [1, 256, 3, 3] | halved again |
| Flatten | [1, 2304] | 256 × 3 × 3 = 2304 |
| Linear + Dropout | [1, 128] → [1, 64] → [1, 10] | dense head |

**Total trainable parameters: ~1.1 million**

Key pattern: as you go deeper, spatial size shrinks and number of filters grows. This compresses spatial information into rich feature representations.

## 2. Famous CNN Architectures (Conceptual Understanding)

### LeNet-5 (1998) — The Original
- Created by LeCun et al., the same person who created MNIST
- One of the first successful CNNs for digit recognition
- Simple stack of Conv → Pool → Conv → Pool → Dense layers
- Historical importance: proved CNNs could work for image recognition

### AlexNet (2012) — The Breakthrough
- By Krizhevsky et al.; won the ImageNet competition with a huge margin
- Top-5 error: 17% vs 26% from second place — a massive leap
- **Key innovation: stacking multiple conv layers directly on top of each other without a pooling layer in between.** This lets the network learn more complex features before downsampling.
- Also used ReLU, Dropout, and GPU training — practices we still use today

### GoogLeNet / Inception (2015) — Multi-Scale Processing
- By Szegedy et al.; top-5 error below 7%
- **Key innovation: the Inception module** — instead of choosing one filter size (e.g., 3×3 or 5×5), it processes the input with multiple filter sizes simultaneously (1×1, 3×3, 5×5) and concatenates the results. This lets the network decide which scale is most useful.
- **1×1 convolutions** — these don't look at spatial neighbors, but they do combine information across channels (depth). They're used to reduce the number of channels cheaply before expensive larger convolutions.
- Was a much deeper network than AlexNet

### ResNet (2015+) — Solving Vanishing Gradients with Skip Connections
- **Key problem it solved:** Very deep networks (50+ layers) were hard to train because gradients vanish as they backpropagate through many layers — early layers stop learning.
- **Key innovation: skip connections (residual connections)** — instead of just learning `h(x)`, the network learns `f(x) = h(x) - x`, so the output is `f(x) + x`. The input is added directly to the output of a block, bypassing some layers.
- This means even if the layers in a block learn nothing (output zero), the input still passes through unchanged — gradients can flow directly backward through the skip connection
- Allows training of extremely deep networks (50, 100, 150+ layers)
- The building block is called a **residual unit**

## 3. Beyond Classification: What CNNs Can Do

### Classification
Standard output: a probability vector over N classes. The model just says "this image is most likely class X."

### Localization
The model doesn't just classify — it also predicts **where** the object is by outputting the coordinates of a **bounding box** (e.g., top-left corner x, y + width + height). This is for a single object of interest.

Think of it as adding a second output head to a classification CNN: one head for the class label, one head for the 4 bounding box coordinates.

### Object Detection
Same idea as localization, but for **multiple objects** in one image. Instead of predicting one bounding box, the model predicts many.

Early approaches just slid a detector across the image at many positions and scales, collecting all boxes where it found something, then removing duplicate overlapping boxes. This was slow but worked.

### Semantic Segmentation
The most detailed task: instead of outputting a single label or a few coordinates, the model classifies **every single pixel** in the image into a category (e.g., sky, road, person, car).

**How it works:** The network first downsamples the image (like a normal CNN) to extract rich features, then **upsamples** it back to the original resolution. Each pixel in the output gets a class label. The upsampling step is the reverse of pooling — it expands a small representation back into a full-sized image.

This is used in self-driving cars (to understand road scenes), medical imaging (to identify tumors pixel by pixel), etc.

## 4. Key Concepts Summary

| Concept | What It Means |
|---------|--------------|
| Stacking conv layers | Learn richer features before pooling (AlexNet innovation) |
| Inception module | Process at multiple scales simultaneously, concatenate results |
| 1×1 convolution | Mixes information across channels without looking at neighbors; used to reduce depth cheaply |
| Skip connection | Add input directly to output of a block; solves vanishing gradients in deep nets |
| Residual unit | The building block of ResNet; learns f(x) = h(x) - x, outputs f(x) + x |
| Localization | CNN with extra output head predicting bounding box coordinates for one object |
| Object detection | Localization extended to multiple objects; remove duplicate overlapping boxes |
| Semantic segmentation | Classify every pixel; requires upsampling back to original image size |
