## PART 1: Training Deep Neural Networks (Continued)

### Transfer Learning
When you need to train a large DNN, don't start from scratch if a pretrained model already exists that does something similar. Instead, **reuse its layers** and only replace the output layer to match your new task.

**How it works:**
1. Take a pretrained model trained on Task A
2. Remove its output layer, add a new one for Task B
3. **Freeze** the reused layers (set `requires_grad=False`) so their weights don't change during early training
4. Train only the new output layer first
5. Gradually **unfreeze** 1–2 top hidden layers and let backprop fine-tune them
6. When unfreezing, **reduce the learning rate** — you don't want to destroy the already-good weights

**Rule of thumb:** The more training data you have, the more layers you can safely unfreeze and retrain.

### Faster Optimizers

Standard SGD just follows the gradient directly. These are smarter alternatives:

**Momentum** — Imagine a ball rolling downhill. Instead of just stepping in the gradient direction, you accumulate speed (momentum) over time. This causes faster convergence. β=0.9 is a typical value.
- Formula: `m ← βm − η∇J(θ)` then `θ ← θ + m`

**Nesterov Accelerated Gradient (NAG)** — Like momentum, but smarter: it measures the gradient slightly *ahead* in the direction momentum is going, which gives a more accurate correction.
- Usually slightly better than plain momentum

**AdaGrad** — Gives each parameter its own learning rate, based on how steep its gradient history is. Steep dimensions get smaller steps; flat dimensions get bigger steps. Good for simple regression, but tends to slow down too much for deep networks.

**RMSProp** — Fixes AdaGrad's problem by only looking at *recent* gradients (uses exponential decay), not all gradients since the beginning. More practical for DNNs.

**Adam** — Combines momentum (tracks past gradients) and RMSProp (tracks past squared gradients). Adaptive learning rate means less tuning needed. Very widely used.

**AdaMax** — A variant of Adam that can be more stable in some cases.

**Nadam** — Adam + Nesterov trick. Sometimes better than Adam.

**AdamW** — Adam + weight decay for regularization. L2 regularization doesn't mix well with standard Adam, so AdamW handles this properly.

---

### Learning Rate Scheduling

The learning rate η controls how big each update step is:
- **Too high** → training diverges (loss explodes)
- **Too low** → training is painfully slow
- **Just right** → fast, stable convergence

The trick: **start with a higher LR and reduce it over time.**

**Exponential Scheduling** — Multiply the LR by a constant γ (e.g., 0.9) every epoch. LR decays smoothly.

**Performance Scheduling (ReduceLROnPlateau)** — Wait `patience` steps. If the metric doesn't improve, multiply LR by `factor`. Adapts to actual training behavior.

**Warmup** — Start with a very small LR, gradually increase it for the first few epochs, then decay. Helps avoid bad early updates when weights are still random.

---

### Regularization (Avoiding Overfitting)

**L2 Regularization (Weight Decay)**
- Penalizes large weights by adding their squared values to the loss
- In PyTorch SGD: just use `weight_decay=1e-4`
- Important: `weight_decay` applies to ALL parameters including bias and batchnorm — which is usually not what you want. Better to implement L2 manually and skip bias/batchnorm terms.

**L1 Regularization**
- Penalizes large weights using absolute values instead of squares
- Tends to produce sparse weights (many zeros)
- Must be implemented manually in PyTorch: `l1_loss = sum(param.abs().sum() for param in params)`

**Dropout**
- During each training step, randomly "drop" (ignore) neurons with probability p
- Forces the network to not rely on any single neuron — knowledge is spread across the network
- At test time, dropout is turned off
- Typical dropout rates:
  - Regular DNNs: 10–50%
  - CNNs: 40–50%
  - RNNs: 20–30%
- Higher dropout = more regularization but slower convergence
- Use `AlphaDropout` instead of `Dropout` when using SELU activations

### Practical Guidelines Summary

These are the recommended default settings when building a DNN:

| Component | Recommendation | Why |
|-----------|---------------|-----|
| Weight Init | He initialization | Works well with ReLU |
| Activation | ReLU (shallow) / Swish (deep) | ReLU is fast; Swish performs better in deep nets |
| Normalization | None if shallow / Batch-norm or Layer-norm if deep | Helps stabilize deep training |
| Regularization | Early stopping + weight decay if needed | Simple and effective |
| Optimizer | Nesterov or AdamW | Best general-purpose choices |
| LR Schedule | Performance scheduling or 1cycle | Adapts to training progress |

## PART 2: Convolutional Neural Networks (CNNs)

### Why CNNs for Images?
A regular MLP applied to a 100×100 image would need to connect every neuron to all 10,000 pixels — that's an enormous number of parameters. CNNs solve this with two ideas:
1. **Partial connections** — neurons only connect to a small local region of the image
2. **Weight sharing** — all neurons in a layer share the same filter weights

This makes CNNs much more efficient and also more effective at detecting spatial patterns.

**Biological inspiration:** Neurons in the cat/monkey visual cortex respond only to small local regions (receptive fields), and some only to specific orientations (e.g., horizontal lines). CNNs mimic this.

---

### Convolutional Layers

Each neuron in a convolutional layer is only connected to a small **receptive field** in the previous layer — not every neuron.

**Key properties:**
- Layers are **2D** (height × width), unlike dense layers which are 1D
- Early layers detect **low-level features** (edges, corners, textures)
- Deeper layers detect **high-level features** (shapes, objects, faces)
- No flattening needed until the very end (before the dense output layer)

**Zero-padding** — Adding zeros around the border of the input so the output layer stays the same size as the input. Without padding, the output shrinks every layer.

**Stride** — How many pixels the receptive field moves each step. A stride of 1 means move one pixel at a time (output is large). A stride of 2 means skip every other pixel (output is about half the size). Think of stride as the "step size" of the sliding window.

---

### Filters (Kernels)

A **filter** (also called a kernel or convolutional kernel) is a small matrix of weights that slides across the entire image.

- Every neuron in a feature map shares the **same filter weights** — this is weight sharing
- A **vertical filter** enhances vertical edges/lines in the image
- A **horizontal filter** enhances horizontal edges/lines
- A **feature map** is the output produced by applying one filter across the whole image — it highlights where in the image that filter's pattern appears most strongly
- A single convolutional layer typically learns **many different filters** (e.g., 32 or 64), producing one feature map per filter
- **The CNN's job during training is to learn the best filters automatically** — you don't design them by hand

# Dropout in a model

# Manual L2 (selective — skips bias and batchnorm)
params_to_reg = [p for name, p in model.named_parameters() if "bias" not in name]
