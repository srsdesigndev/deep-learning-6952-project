## 1. Why RNNs? The Problem with Standard Networks on Sequences

Standard feedforward networks (like the ones we've built so far) take a fixed-size input and produce a fixed-size output. They have no memory of what came before. This works fine for images or tabular data, but fails for sequential data like:
- Time series (stock prices, ridership counts, sensor readings)
- Text / sentences
- Audio

**The key insight:** For sequences, the current value often depends on previous values. A network needs to remember past inputs to make good predictions.

---

## 2. Recurrent Neurons and Layers

A **recurrent neuron** is like a regular neuron, but it also receives its own output from the previous time step as an additional input. This creates a loop — the network feeds information forward in time.

**At each time step t, the neuron receives:**
1. The current input `x(t)` — the new data point
2. Its own previous output `ŷ(t-1)` — what it output last step

**Output formula:**
```
ŷ(t) = f( W_X^T * x(t)  +  W_Ŷ^T * ŷ(t-1)  +  b )
```

Breaking this down:
- `W_X` — weights applied to the current input (like a normal neuron)
- `W_Ŷ` — weights applied to the previous output (the "memory" weights)
- `b` — bias
- `f` — activation function (e.g., tanh)

So each neuron has **two sets of weights**: one for the current input, one for its own past output.

**Unrolling through time:** You can think of an RNN as a very deep network where each "layer" is the same network at a different time step. This is how it's visualized and how it's trained.

---

## 3. Memory Cells

The part of the network that preserves information across time steps is called a **memory cell**.

- The cell's **hidden state** at time t: `h(t) = f(x(t), h(t-1))` — it depends on the current input AND the previous state
- The output `ŷ(t)` usually comes from the current state
- Basic recurrent neurons can only remember short-term patterns — roughly **~10 time steps back**
- More advanced cells (like LSTMs and GRUs) can capture **much longer dependencies**

Think of it like reading a sentence: to understand the word "it" at the end, you need to remember what "it" referred to several words earlier. A basic RNN struggles with this; LSTM/GRU handle it better.

---

## 4. Input and Output Sequence Types

RNNs are flexible — they can be set up in different configurations depending on the task:

**Sequence-to-Sequence**
- Input: a sequence; Output: a sequence (same length)
- Example: Given the last N days of bus ridership, predict ridership for each of those days shifted by one day
- Use case: time series forecasting where you want a prediction at every step

**Sequence-to-Vector**
- Input: a sequence; Output: a single value (only the last output is used, the rest are ignored)
- Example: Read a whole sentence, output a sentiment score (positive/negative)
- Use case: classification or regression on sequences

**Vector-to-Sequence**
- Input: a single vector (repeated); Output: a sequence
- Example: Input an image, output a caption word by word
- Use case: image captioning

**Encoder-Decoder**
- Two RNNs chained together: a **Sequence-to-Vector encoder** followed by a **Vector-to-Sequence decoder**
- The encoder reads the input sequence and compresses it into a single context vector (its final hidden state)
- The decoder takes that context vector and generates the output sequence
- Example: Machine translation — encode an English sentence, decode into French
- This is the foundation of modern NLP models

## 5. Training RNNs — Backpropagation Through Time (BPTT)

To train an RNN, you unroll it through time (like a deep network with one layer per time step), then apply standard backpropagation through all those unrolled steps. This is called **Backpropagation Through Time (BPTT)**.

**The challenge:** Gradients have to flow backward through many time steps. In long sequences, gradients can either:
- **Vanish** — become so small that early time steps learn nothing (the most common problem)
- **Explode** — become so large that training is unstable (less common; solved with gradient clipping)

This is why basic RNNs struggle with long sequences, and why LSTMs were invented.

## 6. Time Series Concepts (From the Ridership Example)

The lecture used Chicago Transit Authority ridership data (daily bus and rail counts) to illustrate these ideas. Know these vocabulary terms:

**Univariate time series** — only one variable tracked over time (e.g., just bus ridership)

**Multivariate time series** — more than one variable tracked over time (e.g., both bus and rail ridership simultaneously)

**Seasonality** — a repeating pattern at a regular interval. The ridership data shows **weekly seasonality** — the same pattern repeats every 7 days (weekdays are high, weekends are low).

**Naïve forecasting** — the simplest possible baseline: predict that tomorrow's value = the value from exactly one period ago (e.g., use last week's ridership as this week's prediction). Always a good starting point to compare against.

**Differencing** — computing the difference between a time series and its lagged version (e.g., `value(t) - value(t-7)`). This shows how much the series changed compared to the same point last week. Useful for removing seasonality.

**Autocorrelation** — when a time series is correlated with a lagged version of itself. If last week's ridership predicts this week's ridership well, the data is autocorrelated. High autocorrelation means naïve forecasting will work reasonably well.

**Evaluation metrics for time series:**
- **MAE (Mean Absolute Error)** — average absolute difference between predictions and true values. Easy to interpret in raw units (e.g., "off by 44,000 riders") but hard to compare across series with different scales.
- **MAPE (Mean Absolute Percentage Error)** — average percentage error. Better for comparing across series. The ridership example showed ~8% error for bus, ~9% for rail.

## 7. Key Concepts Summary

| Concept | What It Means |
|---------|--------------|
| Recurrent neuron | Takes current input + its own previous output as inputs |
| Two weight sets | W_X for current input, W_Ŷ for previous output |
| Memory cell | The part that maintains state across time steps |
| Hidden state h(t) | Depends on current input and previous state: `h(t) = f(x(t), h(t-1))` |
| Short-term memory | Basic RNNs only remember ~10 steps; LSTMs/GRUs do much better |
| BPTT | How RNNs are trained: unroll through time, then backpropagate |
| Vanishing gradient | Gradients shrink to zero over many time steps; main training problem |
| Seq-to-Seq | Input sequence → output sequence (one output per input step) |
| Seq-to-Vec | Input sequence → single output (only last step output used) |
| Vec-to-Seq | Single input → output sequence (input repeated at each step) |
| Encoder-Decoder | Seq-to-Vec encoder + Vec-to-Seq decoder; used for translation |
| Seasonality | Repeating pattern at a fixed interval (e.g., weekly) |
| Naïve forecasting | Use last period's value as prediction; simple but useful baseline |
| Differencing | Value(t) - Value(t-lag); removes seasonality, shows change |
| Autocorrelation | Series is correlated with its own lagged version |
| MAE | Mean absolute error — raw unit error |
| MAPE | Mean absolute percentage error — relative error, easier to compare |

## 8. PyTorch Reference

```python
# Basic RNN layer
rnn = nn.RNN(input_size, hidden_size, num_layers=1, batch_first=True)

# LSTM (better for long sequences — handles vanishing gradient)
lstm = nn.LSTM(input_size, hidden_size, num_layers=1, batch_first=True)

# GRU (simpler than LSTM, often similar performance)
gru = nn.GRU(input_size, hidden_size, num_layers=1, batch_first=True)

# Forward pass (batch_first=True means input shape is [batch, seq_len, features])
output, hidden = rnn(x)        # for RNN/GRU
output, (h_n, c_n) = lstm(x)  # for LSTM (has cell state too)

# Gradient clipping (prevents exploding gradients during BPTT)
nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**Input shape:** `[batch_size, sequence_length, num_features]` (when `batch_first=True`)
**Output shape:** `[batch_size, sequence_length, hidden_size]` — one output per time step
**For Seq-to-Vec:** take only `output[:, -1, :]` — the last time step's output
