## Question 1

**The portion of a sequence-to-vector-to-sequence RNN that "reconstructs" a representation back into a sequence is called the decoder. — True or False?**

- **True** ✅
- False

### ✅ Answer: True

In a seq2vec2seq RNN architecture, the network has two main parts:

- **Encoder** — reads the input sequence step by step and compresses it into a fixed-size vector (the "context vector").
- **Decoder** — takes that fixed vector and **reconstructs** (generates) an output sequence from it, one step at a time.

So the **decoder** is the component responsible for turning the vector back into a sequence. This is foundational in tasks like machine translation, text summarization, and speech recognition.

## Question 2

**Which term do we use to describe consistent, repeating behaviors in time series data?**

- A) Autocorrelation
- B) Cyclic Behavior
- C) Periodicity
- D) **Seasonality** ✅
- E) Trend

### ✅ Answer: D — Seasonality

**Seasonality** describes consistent, repeating patterns that occur at **regular, predictable intervals** (daily, weekly, monthly, yearly).

| Term | What it means |
|---|---|
| **Autocorrelation** | Correlation of a variable with its own past values — a statistical measure, not a pattern name |
| **Cyclic Behavior** | Rises and falls with **no fixed period** (e.g., economic cycles) — irregular timing |
| **Periodicity** | General math term for any repeating pattern — seasonality is the domain-specific term used in time series |
| **Trend** | Long-term upward or downward movement — not a repeating pattern |
| **Seasonality** ✅ | Consistent, repeating pattern tied to a **fixed calendar or time period** |

## Question 3

**Which type of network would be best-suited for taking in a set of nutritional values (e.g., calories, protein, fat) and generating a recipe aimed for those targets?**

- A) Vector-to-Vector
- B) Sequence-to-Vector
- C) Sequence-to-Vector-to-Sequence
- D) **Vector-to-Sequence** ✅
- E) Sequence-to-Sequence

### ✅ Answer: D — Vector-to-Sequence

- **Input:** Nutritional values (calories, protein, fat) = a **fixed-size vector** — no order, no sequence.
- **Output:** A recipe = a **sequence** of ingredients/instructions generated step by step.

This maps perfectly to **Vector-to-Sequence**: fixed vector in → sequence out.

| Option | Why It Doesn't Fit |
|---|---|
| **Vector-to-Vector** | Output would also be a fixed vector, not a recipe |
| **Sequence-to-Vector** | Input must be a sequence — ours is a vector |
| **Seq-to-Vec-to-Seq** | Input must be a sequence (e.g., one recipe → another) |
| **Sequence-to-Sequence** | Both input AND output are sequences — our input is just a vector |
| **Vector-to-Sequence** ✅ | Fixed vector in → generated sequence out — perfect match |

## Question 4

**What types of input data is a recurrent neural network especially well-suited for? Select all (3) that apply.**

- A) Image Data
- B) **Sound Data** ✅
- C) Nominal Data
- D) **Text Data** ✅
- E) Ordinal Data
- F) **Time Series Data** ✅
- G) Tabular Data
- H) Categorical Data

### ✅ Answer: B, D, F — Sound Data, Text Data, Time Series Data

RNNs are built for **sequential data** — where the **order and context** of elements matter.

| Data Type | Why RNNs Excel |
|---|---|
| **Sound Data** ✅ | Audio is temporal — each moment depends on what came before |
| **Text Data** ✅ | Language is sequential — words get meaning from surrounding context |
| **Time Series Data** ✅ | Data points are ordered over time with dependencies between steps |

**Why the others don't fit:**

| Data Type | Why Not |
|---|---|
| **Image Data** | Better handled by CNNs — images have spatial, not sequential structure |
| **Nominal Data** | Categorical labels with no order or sequence |
| **Ordinal Data** | Has ranking but no temporal/sequential dependency |
| **Tabular Data** | Independent rows — feedforward networks or tree models work better |
| **Categorical Data** | Same as nominal — no sequential structure |

## Question 5

**What does it mean to *lag* a set of time series data in a visualization?**

- A) Depicting it with reordered data points based on magnitude instead of time
- B) Depicting it with a rescaled x-axis to compress earlier observations
- C) Depicting it alongside data values from another category
- D) **Depicting it alongside earlier values of the same category** ✅
- E) Depicting it alongside the category's average over a span of time

### ✅ Answer: D — Depicting it alongside earlier values of the same category

**Lagging** a time series means shifting the data in time so you can compare each current value against a **previous (earlier) value of the same variable**. This is used to detect **autocorrelation** — how much a variable is related to its own past.

> Example: Lag by 7 days → each point is plotted next to the value from 7 days prior → reveals weekly patterns.

| Option | Why It's Wrong |
|---|---|
| **A — Reorder by magnitude** | That's sorting, not lagging — lagging preserves time order |
| **B — Rescale x-axis** | Just a formatting change, unrelated to lagging |
| **C — Alongside another category** | That's comparing *different* variables, not a lag |
| **E — Alongside the average** | That describes a **moving average**, not a lag |
