class SimpleClassifierNN(nn.Module):
    def __init__(self, input_dim):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(input_dim, 64),
            nn.ReLU(),
            nn.Linear(64, 1),
        )

    def forward(self, x):
        return self.network(x).squeeze()


def train_classifier(model, X_train, y_train, epochs=20, lr=0.001):
    # Class weight to handle 86/14 imbalance
    pos_weight = torch.tensor([(y_train == 0).sum() / (y_train == 1).sum()])

    dataset = TensorDataset(
        torch.FloatTensor(X_train),
        torch.FloatTensor(y_train.values),
    )
    loader = DataLoader(dataset, batch_size=512, shuffle=True)
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    scheduler = torch.optim.lr_scheduler.StepLR(optimizer, step_size=5, gamma=0.5)
    loss_fn = nn.BCEWithLogitsLoss(pos_weight=pos_weight)
    loss_history = []

    model.train()
    for epoch in range(epochs):
        epoch_loss = 0
        for X_batch, y_batch in loader:
            optimizer.zero_grad()
            loss = loss_fn(model(X_batch), y_batch)
            loss.backward()
            optimizer.step()
            epoch_loss += loss.item()

        avg_loss = epoch_loss / len(loader)
        loss_history.append(avg_loss)
        print(f"Epoch {epoch+1:02d}/{epochs} - Loss: {avg_loss:.4f}")
        scheduler.step()

    return model, loss_history

RQ1 — Diabetes Classification
Dataset: 253,680 samples, severely imbalanced — 86.1% no diabetes, 13.9% diabetes. No missing values.
Logistic Regression (Baseline): Accuracy of 86%, but diabetes recall was only 0.16 — it essentially learned to predict "no diabetes" almost every time. Useless as a medical screening tool.
Simple NN: A big improvement after adding BCEWithLogitsLoss with class weighting. Diabetes recall jumped to 0.66 with precision 0.36 and F1 of 0.46. Overall accuracy dropped to 79% but that's expected — the model is now actually trying to catch diabetic cases.
Deep NN: Nearly identical to Simple NN (recall 0.64, F1 0.46). Adding more layers didn't meaningfully help here, likely because the dataset is tabular and not complex enough to benefit from extra depth.
Threshold sweep: At threshold −1.0, recall hits 0.933+ across both NNs. That's the medically relevant operating point — maximizing recall (catching diabetics) at the cost of precision, which aligns perfectly with your proposal's stated priority.
Confusion matrix (Deep NN at default threshold): 4,546 true positives, 2,523 false negatives, 7,977 false positives. The false negative count is the key number to drive down.

RQ2 — BMI Regression
All three models performed similarly and honestly — weakly.

R² of ~0.16 means the models explain only about 16% of BMI variance. The scatter plot confirms this — predictions cluster tightly around 25–35 regardless of actual BMI, meaning the model is essentially predicting the mean. The log transform didn't help either (R² actually dropped vs. the plain Deep NN).
Why: BMI is likely too weakly correlated with the available lifestyle features to be predicted accurately. The dataset doesn't include key BMI predictors like caloric intake, genetics, or detailed activity data.

Key Takeaways

Classification is the stronger task. With proper class weighting and threshold tuning, the NNs achieve clinically useful recall for diabetes detection.
Regression is the weaker task. Low R² across all models suggests BMI isn't well-explained by the features available — worth acknowledging as a limitation.
Training loss curves show both NNs converging cleanly without divergence, confirming stable training. The Deep NN converged slower on classification but reached the same final loss as Simple NN.
Model complexity didn't help much on either task — the simple and deep NNs are nearly tied, suggesting the bottleneck is the data, not the architecture.
